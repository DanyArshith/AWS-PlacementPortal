# College Placement Portal — Manual Services Integration Checklist

This checklist outlines the exact manual configurations you must perform in the AWS Console, MongoDB Atlas, and your code repositories to integrate all placement portal services (Frontend, Backend, Database, S3 Uploads, Nginx, CloudFront CDN, Route53 DNS, and CloudWatch Monitoring).

---

## Phase 1: Database Setup (MongoDB Atlas)

To persist users, jobs, applications, and student profiles:

- [ ] **Create a MongoDB Atlas Project**: Log in to MongoDB Atlas and create a new project.
- [ ] **Provision a Cluster**: Deploy a shared tier cluster (M0 - Free Tier is sufficient) in your preferred region.
- [ ] **Configure Network Access**:
  - Go to **Network Access** -> **Add IP Address**.
  - For staging/initial deploy, whitelist `0.0.0.0/0` (Access from anywhere).
  - *Best Practice*: Once your EC2 instance is created and has an Elastic IP, replace `0.0.0.0/0` with the EC2 Instance's IP `/32` to restrict database connection access to just your app server.
- [ ] **Create Database User**:
  - Go to **Database Access** -> **Add New Database User**.
  - Choose authentication method **Password**.
  - Set a username (e.g. `portal_user`) and a secure password. Keep these credentials safe.
  - Assign the role **Read and write to any database**.
- [ ] **Retrieve Connection String**:
  - Go to **Database -> Clusters**, click **Connect**.
  - Select **Drivers** (Node.js).
  - Copy the connection string. It will look like:
    `mongodb+srv://portal_user:<password>@cluster0.abcde.mongodb.net/?retryWrites=true&w=majority&appName=Cluster0`
  - Replace `<password>` with your database user's password. Note this string down for your backend `.env` configuration.
- [ ] **Seed Default Admin Account**:
  - Locally (or on the server), edit the backend `.env` to include this connection string under `MONGODB_URI`.
  - Run the seed script:
    ```bash
    cd /path/to/placement-portal/backend
    node scripts/seedAdmin.js
    ```
  - Verify that it outputs: `Admin user seeded successfully`.

## Phase 2: Secure Storage Setup (Amazon S3)

We need one bucket for hosting the public static website frontend, and another private bucket for storing student resumes/photos securely.

### Bucket A: Resume & Photo Storage (Private)
- [ ] **Create Bucket A**: Name it uniquely (e.g., `placement-portal-resumes-prod`).
- [ ] **Block Public Access**: Keep **Block *all* public access** checked. Files are private and only read/written via secure presigned URLs generated by the backend server.
- [ ] **Set S3 CORS**:
  - Under the **Permissions** tab of the bucket, edit **Cross-origin resource sharing (CORS)**.
  - Paste the following json to allow your domain origins to request files:
    ```json
    [
      {
        "AllowedHeaders": ["*"],
        "AllowedMethods": ["GET", "PUT", "POST", "DELETE", "HEAD"],
        "AllowedOrigins": ["*"],
        "ExposeHeaders": []
      }
    ]
    ```

### Bucket B: Static Website Hosting (Public)
- [ ] **Create Bucket B**: Name it uniquely (e.g., `placement-portal-frontend-prod`).
- [ ] **Uncheck Block Public Access**: Clear **Block *all* public access** to allow public website visitors to read HTML/CSS/JS files.
- [ ] **Enable Static Website Hosting**:
  - Under the **Properties** tab, scroll to **Static website hosting** and click **Edit**.
  - Enable it and specify `index.html` as both the **Index document** and **Error document**.
  - Click **Save**. Note down the generated **Bucket website endpoint** (e.g., `http://your-bucket.s3-website.amazonaws.com`).
- [ ] **Set Bucket Policy**:
  - Under the **Permissions** tab, edit **Bucket policy** and paste:
    ```json
    {
      "Version": "2012-10-17",
      "Statement": [
        {
          "Sid": "PublicReadGetObject",
          "Effect": "Allow",
          "Principal": "*",
          "Action": "s3:GetObject",
          "Resource": "arn:aws:s3:::your-bucket-name/*"
        }
      ]
    }
    ```
    *(Replace `your-bucket-name` with your actual Bucket B name)*.

## Phase 3: Access Control Setup (IAM Roles)

To allow the EC2 server to communicate with S3 and CloudWatch without hardcoding keys:

- [ ] **Create IAM Policy for S3**:
  - Navigate to **IAM Console -> Policies -> Create policy**.
  - Select JSON editor and paste the policy from [deploy_monitoring.md](file:///d:/Projects/AWS/docs/deploy_monitoring.md) (restricting actions on Bucket A).
  - Name it `PlacementPortalS3AccessPolicy`.
- [ ] **Create IAM Role**:
  - Navigate to **IAM Console -> Roles -> Create role**.
  - Choose Trusted Entity: **AWS Service**, Service: **EC2**.
  - Search and select two policies to attach:
    1. `PlacementPortalS3AccessPolicy` (created in previous step).
    2. `CloudWatchAgentServerPolicy` (AWS-managed policy).
  - Name the role `PlacementPortalEC2Role` and create it.

## Phase 4: Compute Setup (EC2 Instance)

- [ ] **Launch EC2 Instance**:
  - Select **Ubuntu Server 22.04 LTS (HVM)**.
  - Instance Type: `t2.micro` or `t3.micro`.
  - Configure Key Pair for SSH.
- [ ] **Configure Security Group**:
  - Create a group `placement-portal-backend-sg` with rules:
    - SSH (Port 22) -> restricted to `My IP`.
    - HTTP (Port 80) -> `0.0.0.0/0`.
    - HTTPS (Port 443) -> `0.0.0.0/0`.
- [ ] **Allocate Elastic IP (EIP)**:
  - Navigate to **EC2 -> Network & Security -> Elastic IPs**.
  - Click **Allocate Elastic IP address**.
  - Select the allocated IP, click **Actions -> Associate Elastic IP address**, and link it to your EC2 instance.
- [ ] **Attach IAM Role to EC2**:
  - Select your instance in the EC2 Console.
  - Click **Actions -> Security -> Modify IAM Role**.
  - Select `PlacementPortalEC2Role` and save.

## Phase 5: Decoupled Integration & Code Configuration

Now connect the API server IP/domain and S3 settings in your codebases.

- [ ] **Configure Backend Environment Variables (`.env`)**:
  - SSH to your EC2 instance. Create `/var/www/placement-portal/backend/.env` containing:
    ```ini
    PORT=4000
    NODE_ENV=production
    MONGODB_URI=your_mongodb_atlas_connection_string
    JWT_SECRET=your_long_cryptographic_secret_string
    S3_BUCKET_NAME=placement-portal-resumes-prod
    AWS_REGION=your_aws_region_code (e.g. us-east-1)
    ```
- [ ] **Configure Frontend API Endpoint**:
  - On your local development machine, open [api.js](file:///d:/Projects/AWS/placement-portal/frontend/js/api.js).
  - Update `API_BASE` (line 16) to point to your backend domain or Elastic IP:
    `const API_BASE = 'https://api.yourdomain.com/api';`
    *(For testing without a domain, use: `http://<your-ec2-elastic-ip>/api`)*.
- [ ] **Compile and Upload Frontend**:
  - Compile the static assets locally:
    ```bash
    npm run build
    ```
  - Upload the contents of the `./dist` folder (excluding the folder itself) into **Bucket B** via the AWS Console or using the AWS CLI:
    ```bash
    aws s3 sync dist/ s3://placement-portal-frontend-prod --delete
    ```

## Phase 6: Server Setup & SSL (Nginx & PM2)

- [ ] **Deploy Code to EC2 & Run Backend**:
  - Pull your latest code on EC2.
  - Install backend dependencies and launch PM2:
    ```bash
    cd /var/www/placement-portal/backend
    npm install --omit=dev
    pm2 start ecosystem.config.js --env production
    pm2 startup
    # (Execute the command outputted by pm2 startup to save it to systemd)
    pm2 save
    ```
- [ ] **Configure Nginx Reverse Proxy**:
  - Create config `/etc/nginx/sites-available/placement-portal`.
  - Add the reverse proxy blocks (proxying traffic to `http://127.0.0.1:4000`) detailed in [deploy_backend.md](file:///d:/Projects/AWS/docs/deploy_backend.md).
  - Symlink to `sites-enabled/`, delete `default`, run `nginx -t` to test, and run `sudo systemctl restart nginx`.
- [ ] **Install SSL/TLS Certificate**:
  - If using a domain mapped to your EC2 Elastic IP:
    ```bash
    sudo certbot --nginx -d api.yourdomain.com
    ```
  - Accept redirection configurations.

## Phase 7: Edge Distribution & DNS (CloudFront & Route53)

To ensure secure static website access and high load performance:

- [ ] **Create CloudFront Distribution**:
  - Origin Domain: Enter the **Static Website Hosting Endpoint** (e.g. `placement-portal-frontend-prod.s3-website-us-east-1.amazonaws.com`). Do not select the bucket ARN dropdown.
  - Viewer Protocol Policy: **Redirect HTTP to HTTPS**.
  - Click **Create**.
- [ ] **Point DNS A Records in Route53**:
  - Go to your Hosted Zone in Route53.
  - Create an **A Record** (e.g. `yourdomain.com`).
  - Toggle **Alias** -> **Enable**.
  - Route traffic to **Alias to CloudFront distribution** and select your distribution.

## Phase 8: Logging & Monitoring (CloudWatch Setup)

- [ ] **Install and Start CloudWatch Agent**:
  - Follow the setup commands in [deploy_monitoring.md](file:///d:/Projects/AWS/docs/deploy_monitoring.md) to install the deb package on Ubuntu.
  - Run the agent-ctl binary targeting `backend/aws/amazon-cloudwatch-agent.json`.
- [ ] **Configure Alarms**:
  - Go to the **CloudWatch Console -> Log groups**.
  - Select your log group `placement-portal-backend-error`. Create a metric filter `{ $.level = "error" }`.
  - Link this metric to a new Alarm. Set the threshold to `>= 5` events in `5 minutes`.
  - Set action to publish to a standard **SNS Topic** subscribed to your email.
